# 정처기 실기 요약_11과목_응용 SW 기초 기술 활용

## 운영체제
사용자가 컴퓨터 하드웨어를 쉽게 사용할 수 있도록 인터페이스를 제공해주는 소프트웨어<br>
인터페이스(쉘) + 커널 구조

<br>

**특징**<br>
사용자 편리성, 인터페이스, 스케줄링, 자원관리, 제어 기능

<br>

**쉘**<br>
사용자가 입력한 명령어 라인을 읽어 필요한 시스템 기능을 실행하는 명령어 해석기

<br>

**커널**<br>
운영체제에서 핵심 기능이 모여있는 프로그램

<br>

**커널의 특징**<br>
프로세스 관리, 기억장치 관리, 주변장치 관리, 파일 관리

<br>

## 운영체제 종류<br>
윈도즈, 유닉스, 리눅스, 맥, 안드로이드

<br>

**윈도즈 특징**
- GUI 제공
- 선점형 멀티태스킹 방식 제공
- 자동감지 기능 제공
- OLE 사용

<br>

**유닉스 특징**
- 대화식 운영체제 기능 제공
- 다중 작업 기능 제공
- 다중 사용자 기능 제공
- 이식성 제공
- 계층적 트리 구조 파일 시스템 제공
  
<br>

**리눅스**<br>
유닉스 기반으로 개발된 오픈 소스 기반 운영체제이며 유닉스의 특징과 같음

<br>

<br>

**CLI**<br>
사용자가 직접 명령어를 입력하는 방식

<br>

**GUI**<br>

## 운영체제 핵심 기능
메모리 관리, 프로세스 관리

<br>

**메모리 관리 기법(반배할교)**
- 반입기법 : 메모리로 적재 시기 결정
- 배치기법 : 메모리 적재 위치 결정
- 할당기법 : 메모리 적재 방법 결정
- 교체기법 : 메모리 교체 대상 결정

<br>

**메모리 배치 기법**
- 최초 적합 : 가용 공간 중 첫 번째 분할에 할당
- 최적 적합 : 가장 크기가 비슷한 공간에 할당
- 최악 적합 : 가장 큰 공간에 할당

<br>

**메모리 할당 기법(연단다, 분페세)**
- 연속 할당 기법 : 주기억장치 내에서 인접되게 연속하여 저장
- - 단일 분할 할당 기법
- - 다중 분할 할당 기법
- 분산 할당 기법 : 하나의 프로세스를 여러개로 나누어 분산 배치
- - 페이징 기법
- - 세그먼테이션 기법

<br>

**교체기법**<br>
FIFO, LRU, LFU, OPT, NUR, SCR

## 프로세스
CPU에 의해 처리되는 실행중인 프로그램을 의미하고 작업, 태스크라고도 한다

<br>


**프로세스 상태(생준실대완)**
- 생성상태 : 프로세스가 생성된 상태
- 준비상태 : CPU를 할당받을 수 있는 상태
- 실행상태 : CPU를 할당받아 동작 중인 상태
- 대기상태 : 처리 완료까지 대기하는 상태
- 완료상태 : 완전히 수행을 종료한 상태

<br>

**프로세스 상태 전이**
- 디스패치 : 준비 상태에서 실행 상태로 전이
- 타이머 런 아웃 : 실행 상태에서 준비 상태로 전이
- 입출력 발생 : 실행 상태에서 대기 상태로 전이
- 웨이크 업 : 대기 상태에서 준비 상태로 전이

<br>

**프로세스 스케줄링 유형**
- 선점형 스케줄링<br>
하나의 프로세스가 CPU를 차지하고 있을 때, 우선순위가 높은 다른 프로세스가<br>
현재 프로세스를 중단시키고 CPU를 점유하는 방식

- 비선점형 스케줄링<br>
한 프러세스가 CPU를 할당받으면 작업 종료 후 CPU 반환 시까지 다른 프로세스는<br>
CPU점유가 불가능한 방식

<br>

**선점형 스케줄링 알고리즘**
- 라운드 로빈 : 균등하게 CPU 점유시간을 할당
- SRT : 빨리 끝나는 프로세스를 먼저 수행
- 다단계 큐 : 여러 그룹으로 분할한 큐를 이용해서 상위단계 작업에 의한 하위단계 작업이 선점 당함
- 다단계 피드백 큐 : 큐마다 서로 다른 CPU시간 할당, FIFO + 라운드 로빈 혼합

<br>

**비선점형 스케줄링 알고리즘**
- 우선순위 : 프로세스별로 우선순위 할당
- 기한부 : 기간이나 시간 내에 완료되도록 계획
- FCFS : 대기큐에 도착한 순서에 따라 CPU할당
- SJF : 프로세스가 도착한 시점 당시에 가장 작은 서비스 시작을 갖는 프로세스가 자원을 점유
- HRN : 대기 프로세스중 응답률이 가장 높은 것을 선택

<br>

## 교착상태
다중프로세싱 환경에서 두 개 이상의 프로세스가 자원할당을 무한 대기하는 상태

<br>

**교착상태 발생 조건(상점비환)**<br>
상호배제, 점유와 대기, 비선점, 환형 대기

<br>

**교착상태 해결 방법(예회발복)**<br>
예방, 회피, 발경, 복구

<br>

## 디스크 스케줄링 종류
FCFS, SSTF, SCAN, C-SCAN, LOOK, N-STEP SCAN, SLTF

<br>

## 가상화
물리적인 리소스를 하나로 보이게 하거나, 여러개로 보이게 하는 기술<br>
대부분 서버는 옹량의 20%를 사용하는데 가상화로 가동률을 60~70%로 올릴 수 있음

<br>

**가상화 종류**<br>
- 플랫폼 가상화 : 호스트 프로그램이 게스트 프로그램을 만들어서 독립된 환경을 만들어낸 것 처럼 보여주는 기법
- 리소스 가상화 : 게스트 소프트웨어에서 사용자는 독립된 하드웨어에서 소프트웨어가 실행되는 것처럼 활용하는 기법

<br>

**가상화 기술요소**<br>
컴퓨팅 가상화, 스토리지 가상화, I/O 가상화, 컨테이너, 분산처리 기술, 네트워크 가상화 기술

<br>

## 클라우드 컴퓨팅
인터넷을 통해 가상화된 컴퓨터 시스템 리소스를 제공하고, 정보를 자신의 컴퓨터가 아닌 클라우드에 연결된 다른 컴퓨터로 처리하는 기술

<br>

**클라우드 컴퓨팅 분류(사공하)**<br>
사설 클라우드, 공용 클라우드, 하이브리드 클라우드

<br>

**클라우드 컴퓨팅 유형**<br>
인프라형 서비스, 플랫폼형 서비스, 소프트웨어형 서비스

<br>

## 네트워크
- LAN(근거리 통신망) : 작은 지역을 커버하는 네트워크
- WAN(원거리 통신망) : LAN보다 전송거리가 넓지만 에러율이 높고 전송 지연이 큼

<br>

## OSI 7계층(국 : 물데네전세표응, 영 : 아파서티내다피나다)
- 응용계층 : 사용자와 네트워크 간 응용서비스 연결
- - 프로토콜 : HTTP, FTP
- - 전송단위 : 데이터
- - 장비 : 호스트(PC)
- 표현계층 : 데이터 형식 설정
- - 프로토콜 : JPEG, MPEG
- - 전송단위 : 데이터
- - 장비 : 호스트(PC)
- 세션계층 : 송수신 간의 논리적 연결
- - 프로토콜 : RPC, NetBIOS
- - 전송단위 : 데이터
- - 장비 : 호스트(PC)
- 전송계층 : 송수신 프로세스 간의 연결과 신뢰성 있는 통신 보장
- - 프로토콜 : TCP, UDP
- - 전송단위 : 세그먼트
- - 장비 : L4스위치
- 네트워크계층 : 단말기 간 데이터 전송을 위한 최적화 경로 제공
- - 프로토콜 : IP, ICMP, ARP, RARP, IGMP, 라우팅
- - 전송단위 : 패킷
- - 장비 : 라우터
- 데이터링크계층 : 노드간 회선제어, 흐름제어, 오류제어 기능 수행
- - 프로토콜 : HDLC, PPP, ATM
- - 전송단위 : 프레임
- - 장비 : 브리지, 스위치
- 물리계층 : 0과 1의 비트 정보를 회선에 보내기 위한 전기전 신호 변환
- - 프로토콜 : RS-232C
- - 전송단위 : 비트
- - 장비 : 허브, 리피터

<br>

## 프로토콜
서로 다른 시스템이나 기기들 간의 데이터 교환을 원활히 하기위한 표준화된 통신규약

**프로토콜 기본 3요소(구의타)**<br>
구문, 의미, 타이밍

<br>

**네트워크 프로토콜**<br>
컴퓨터나 원거리 통신 장비 사이에서 메시지를 주고받는 양식과 규칙의 체계

<br>

**네트워크 프로토콜 특징**<br>
단편화, 재조립, 캡슐화, 연결 제어, 오류 제어, 동기화, 다중화, 주소 지정

<br>

**서브네팅**<br>
IP주소 고갈문제를 해결하기 위해 원본 네트워크를 여러 개의 네트워크로 분리하는 과정

<br>

**서브네팅 방식**<br>
- FLSM : 서브넷 길이를 고정적으로 사용, 한 대역을 동일한 크기로 나눔
- VLSM : 서브넷 길이를 가변적으로 사용, 한 대역을 다양한 크키로 나눔

<br>

## 네트워크 구조

**애드 혹 네트워크**<br>
노드들에 의해 자율적으로 구성되는 기반 구조가 없는 네트워크

<br>

**네트워크 설치 구조**
- 버스형 : 여러 대의 노드가 멀티 포인트로 연결된 구조
- 트리형 : 각 노드가 계층적으로 연결되어 있는 구조
- 링형 : 모든 노드가 하나의 링에 순차적으로 연결되는 구조
- 성형 : 단말 노드가 허브라는 장비에 점 대 점으로 연결된 구조

<br>
